# Contributing to Meshview

## Простые правила внесения изменений
- Коммиты: используйте понятные сообщения в стиле Conventional Commits
- Код: придерживайтесь принятого в проекте стиля и обязательно проходите линтеры и проверки перед PR
- Теги релизов: теги выпускаются на основе версии родительского репозитория

## Версионирование тегов
Формат тега:  rel-X.Y.Z.N.M, где X.Y.Z версия из родительского репозитория поверх которой делали изменения

Например
- rel-3.0.7.0.1 это fix на основе версии 3.0.7 родительского репозитория
- rel-3.0.7.1.0 это feature на основе версии 3.0.7 родительского репозитория
- rel-3.0.7.1.3 это третий фикс на основе версии 3.0.7.1 текущего репозитория
